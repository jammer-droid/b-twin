# B-TWIN 실행 맥락 기록 (P1→P3)

- 작성일: 2026-03-06
- 목적: 대화 맥락 유실 없이, 합의된 단계(P0→P1→P2→P3) 기준으로 순차 실행 상태를 고정

---

## 1) 기준 합의 (고정)

- **P0**: 운영 모드 2종 공식화 / 코어 포트 명세 / 코드 반영 정리
- **P1**: KPI + 정합성 가드 + 리콜 품질 개선
- **P2**: OpenClaw 결합 어댑터 / 단독 런타임 최소 구현 / 운영 대시보드
- **P3**: 통합 운영화(회귀·복구·성능 검증, 릴리스 기준/런북/KPI 리포트 체계 고정)

> 주의: 외부 리서치봇 문서 기반 우선순위가 아니라, 위 합의안을 단일 기준으로 사용한다.

---

## 2) 현재 상태 스냅샷

### 완료됨
- P0 핵심 구현은 mainline에 반영됨
  - runtime mode(attached/standalone)
  - core runtime port 계약 코드 반영
  - API/CLI runtime semantics 검증

### 미구현 시작점
- **P1 전체부터 미착수**로 간주

---

## 3) 순차 실행 작업 목록 (실행 순서 고정)

### P1-1. 동기화 공백 KPI 도입
- 목표
  - `write→indexed` 지연 측정
  - `manifest↔vector mismatch` 건수 측정
  - `repair` 성공률/평균 복구시간 측정
- 완료 기준
  - 지표 수집 코드 + 테스트 + 산출 포맷(CLI/API 또는 로그) 반영

### P1-2. 정합성 가드 강화
- 목표
  - handoff/complete 직전 `indexed + checksum` 검증
  - 실패 시 `repair(doc_id)` 재시도 정책 적용
- 완료 기준
  - 실패/성공 경로 테스트 포함

### P1-3. 리콜 품질 개선
- 목표
  - Hybrid 검색(BM25+vector)
  - MMR(중복 억제)
  - Temporal Decay(최신성 가중)
  - 임베딩 캐시
- 완료 기준
  - 검색 정확도/중복도/응답성 지표에서 개선 확인

### P2-1. OpenClaw 결합 어댑터 구현
- 목표
  - OpenClaw memory_search/get + B-TWIN 저장 규칙 브릿지
  - 공통 감사 이벤트 포맷 정렬

### P2-2. 단독 모드 최소 런타임 구현
- 목표
  - 기본 세션/리콜/권한 모듈로 코어 독립 동작

### P2-3. 운영 대시보드 구축
- 목표
  - 인덱서 상태, 실패 큐, 복구 이력, 게이트 위반 현황 가시화

### P3-1. 통합 검증/운영화
- 목표
  - 회귀·복구·성능 시나리오 검증
  - 릴리스 체크리스트, 운영 Runbook, KPI 주간 리포트 체계 확정

---

## 4) 실행 방식

- **순차 처리**: 위 순서를 반드시 지킨다.
- **서브에이전트 기반**: 작업 단위로 구현 서브에이전트 실행
- **리뷰/수정 반복**
  1. 스펙 적합성 리뷰
  2. 코드 품질 리뷰
  3. 이슈 발생 시 수정 후 재리뷰
