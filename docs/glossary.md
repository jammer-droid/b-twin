# B-TWIN 용어집 v1

본 용어집은 QA 문서, 운영 런북, 리뷰 체크리스트에서 동일한 의미로 사용해야 하는 핵심 용어를 정의한다.

## 핵심 용어

- **정합성(Consistency)**: 실행 결과(로그/이벤트)와 문서 상태(매니페스트/레코드)가 같은 사실을 가리키는 정도
- **무결성(Integrity)**: 상태/문서가 변조·누락 없이 검증 가능한 상태를 유지하는 성질
- **책임성(Accountability)**: 상태 전이에 대해 `누가/무엇을/언제/왜`를 재구성 가능한 성질
- **감사 추적(Audit trail)**: 상태 변화/승인/복구 이력을 append-only 방식으로 남긴 증적
- **동기화 공백(sync gap)**: 실행 완료 시점과 문서 반영 시점 사이의 지연/불일치 구간

## 표준 운영 시퀀스

- `mark_pending`: 미확정 상태를 명시
- `refresh`: 최신 원천 상태 재수집
- `reconcile`: 실행 로그와 문서 상태를 대조해 불일치 계산
- `repair`: 불일치 보정 이벤트 적용
- `checksum`: 보정 결과 무결성 검증
- `doc_version`: 문서 버전 단조 증가 및 변경 책임 추적

## 문서 작성 규칙

1. 첫 등장 시에만 한/영 병기 허용 (예: 정합성(Consistency))
2. 이후에는 한국어 용어 우선 사용
3. 코드 식별자(`mark_pending`, `doc_version` 등)는 코드 표기 유지
4. 동의어 혼용 금지: 동일 문서 내에서 한 용어만 사용
